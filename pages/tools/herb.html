scripty.txt

scriptability

First off, I'm going to note that benefits which herbstluftwm brings. Herbstluftwm is configured via shell script, this means that it is extremely easy to cram your own features into your config file. Herbstluftwm is entirely controlled via shell commands, attributes of windows can be altered via simple herbstclient commands. These can be used to do things like resizing windows or moving them to different workspaces, among many others. Essentially anything you would expect to be able to do with a keybind is doable with an herbstclient command and this is because the keybinds are herbstclient commands, keybinds execute herbstclient commands and are created using them. This is incredibly useful because it means you can script your window manager inside and outside of your config however you please.

herbstluftwm tiling

In terms of tiling concepts, herbstluftwm offers the power of frames. Frames are really cool and form the basis of what makes herbstluftwm tiling "manual". Herbstluftwm frames essentially all act like their own isolated little space, the current workspace may be split vertically or horizontally within the current frame (by default the current workspace will initially behave as a single large frame), once splitting windows may be moved between frames and frames may be resized, although windows cannot be resized (well they kind of can but I'll get into that later, trust me frame resizing usually works well enough that you won't be hurting from losing the ability to resize single windows, it's usually convenient enough to just resize their current frame or make a new one). Within each frame a tiling style is adherred to, by default these are toggled via $mod-Space and there are 4, here's what they all do.

1 + windows are tiled vertically, on top of each other
2 + windows are tiled horizontally, side by side
3 + window space is partitioned to equally distribute both horizontal and vertical space
4 + windows will all stack on top of each other and may be tabbed through
