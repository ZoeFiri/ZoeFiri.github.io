scripty.txt

scriptability

First off, I'm going to note that benefits which herbstluftwm brings. Herbstluftwm is configured via shell script, this means that it is extremely easy to cram your own features into your config file. Herbstluftwm is entirely controlled via shell commands, attributes of windows can be altered via simple herbstclient commands. These can be used to do things like resizing windows or moving them to different workspaces, among many others. Essentially anything you would expect to be able to do with a keybind is doable with an herbstclient command and this is because the keybinds are herbstclient commands, keybinds execute herbstclient commands and are created using them. This is incredibly useful because it means you can script your window manager inside and outside of your config however you please.

tiling.txt

herbstluftwm tiling

in terms of tiling concepts, herbstluftwm offers the power of frames. frames are really cool and form the basis of what makes herbstluftwm tiling "manual". herbstluftwm frames essentially all act like their own isolated little space, the current workspace may be split vertically or horizontally within the current frame (by default the current workspace will initially behave as a single large frame), once splitting windows may be moved between frames and frames may be resized, although windows cannot be resized (well they kind of can but i'll get into that later, trust me frame resizing usually works well enough that you won't be hurting from losing the ability to resize single windows, it's usually convenient enough to just resize their current frame or make a new one). within each frame a tiling style is adherred to, by default these are toggled via $mod-space and there are 4, here's what they all do.

tiling styles

1 + windows are tiled vertically, on top of each other
2 + windows are tiled horizontally, side by side
3 + window space is partitioned to equally distribute both horizontal and vertical space
4 + windows will all stack on top of each other and may be tabbed through


qualities of frames

1 + windows can be freely moved between them
2 + they may be resized
3 + they may be split horizontally or vertically into two smaller frames
4 + they may be merged back into a single frame
5 + they cannot be moved, although you may change their layout by simply merging them back as far as necessary and then splitting again. this has no effect upon the windows contained by the frames.

float.txt

floating

if you ever find yourself needing to float windows herbstluftwm does have minimal float support, this is done with the keybind $mod-s by default although this will make the entire workspace float, this cannot be changed and is simply how herbstluftwm floating mode works. In floating mode windows are resized by right click dragging with mod key and left click to drag. As far as I can tell, herbstluftwm does not support resizing windows with keybinds (just frames, which are not relevant in floating mode). For this, I've written a pretty bad little script but it works, feel free to rip it off. It will also resize "pseudotiled" windows although this does not alter their allocated tile space.

#!/usr/bin/env bash
ID=$(xdotool getactivewindow)
DIMS=$(xwininfo -id $ID)
HEIGHT=$(echo $DIMS | grep Height |sed 's/.*Height: \([0-9]*\).*/\1/g')
WIDTH=$(echo $DIMS | grep Width |sed 's/.*Width: \([0-9]*\).*/\1/g')
[ $1 = "+x" ] && xdotool windowsize $ID $((WIDTH+32)) $HEIGHT
[ $1 = "-x" ] && xdotool windowsize $ID $((WIDTH-32)) $HEIGHT
[ $1 = "+y" ] && xdotool windowsize $ID $WIDTH $((HEIGHT+32))
[ $1 = "-y" ] && xdotool windowsize $ID $WIDTH $((HEIGHT-32))

gettingstarted.txt

getting started

setting up autostart

after initially installing herbstluftwm, in order to configure it you will need to put the "autostart" inside of ~/.config/herbstluftwm/autostart so just run
cp /etc/xdg/herbstluftwm/autostart ~/.config/herbstluftwm/ as a side note, "herbstclient" is aliased to "hc" in the config by default.

modkey

"autostart" is merely a bash script, upon running herbstluftwm (or reloading it) it is executed. One of the first things you'll need to do is set your mod key, by default this is set to Mod1 for Alt but if you want the Win key set it to Mod4.

startup

To setup your normal startup processes, just make a section of your config where they are launched via hc spawn [process]. this command just makes herbstluftwm spawn a process, and is needed as opposed to just typing out the name of the program so that the autostart doesn't hang on it if that program does not return control to shell, which would result in only the configuration up to that point taking effect. For any process which remains open after this, it is recommended that before running these startup lines all of those processes have a corresponding pkill line so that you don't get multiple instances.

keybinds

keybinds in herbstluftwm are added via herbstclient keybind [key1-key2-key(n)] [command] commands. It is important to note that the commands will all be herbstclient commands, for example using xbacklight -inc -10 as the command will pass that command to herbstclient, so it must be ran with the spawn command within the keybind.

basic usage

bind action
mod-up/down/left/right move window focus
mod-shift-up/down/right move window
mod-ctrl-up/down/right resize current frame
mod-o/u split frame horizontal/vertical
mod-r merge frames
mod-(#) go to workspace (#)
mod-shift-(#) send window to workspace (#)
mod-f fullscreen
mod-p pseudotile
mod-s toggle floating
mod-shift-c close window
mod-return spawn terminal
